# 에이전트 3 작업 명세 - 이미지형 키워드·상세 용어해설 강화형 v9 샘플

## 0. 반영 방향

본 문서는 기존 3페이지 구조를 유지하면서, **주요 키워드를 텍스트 태그가 아니라 이미지형 카드로 제시**하고, **용어 정의를 더 전문적이면서도 쉽게 이해할 수 있도록 상세화**한 샘플 화면설계서이다.

이번 버전의 핵심 변경사항은 다음과 같다.

1. 총 3개 페이지 구조 유지
2. LLM은 1페이지에서만 작동
3. 2페이지 사업요약 화면에 **이미지형 핵심 키워드 카드** 추가
4. 키워드 카드는 `아이콘/SVG + 키워드 + 한 줄 설명 + 과업 연결성`으로 구성
5. 용어 정의는 더 길고 쉽게 이해할 수 있게 보완
6. 용어 설명은 `전문 정의 / 쉽게 이해하기 / 과업에서 보는 관점 / 실무 체크포인트`로 구분
7. 신입사원, 타 부서, 사업관리자, 전문가가 모두 볼 수 있는 구조로 설계

## 2. 이미지형 핵심 키워드 카드 정책

### 2.1 카드 구성

각 핵심 키워드는 단순 텍스트 태그가 아니라 카드형 이미지 요소로 제시한다.

```text
┌────────────────────┐
│  아이콘 / SVG       │
│  핵심 키워드         │
│  한 줄 설명          │
│  과업 연결성         │
└────────────────────┘
```

### 2.2 카드 항목

| 항목 | 설명 |
|---|---|
| 아이콘/SVG | 파도, 태풍, 지도, AI 칩, 위성, 데이터베이스 등 시각 요소 |
| 키워드 | 사업 이해에 필요한 핵심 개념 |
| 한 줄 설명 | 누구나 이해할 수 있는 쉬운 설명 |
| 과업 연결성 | 이 키워드가 본 사업에서 어떤 역할을 하는지 표시 |

# 4. 해안침수예상도 제작 - 상세 용어 해설

## 4.1 해안침수예상도

| 구분 | 설명 |
|---|---|
| 전문 정의 | 폭풍해일, 월파, 해수면 상승, 강우, 하천유량, 내수침수 등 다양한 침수 요인을 고려하여 연안지역의 예상 침수범위와 침수심을 지도 또는 공간정보 형태로 표현한 재해위험 정보 |
| 쉽게 이해하기 | 태풍, 높은 파도, 폭우가 왔을 때 해안가 주변의 어느 지역이 얼마나 잠길 수 있는지를 보여주는 위험지도이다. 일반 지도 위에 침수 가능 구역과 깊이를 표시한다고 이해하면 된다. |
| 과업에서 보는 관점 | 본 사업의 최종 핵심 산출물이다. 단순한 그림이 아니라 수치모형 계산, 현장조사, 지형자료, 과거 피해자료 등을 종합해 만든 방재 의사결정 자료이다. |
| 실무 체크포인트 | 도면 축척, GIS 파일 형식, 침수심 구간, 빈도별 시나리오, 지자체 활용 가능성을 확인해야 한다. |

## 4.2 복합침수

| 구분 | 설명 |
|---|---|
| 전문 정의 | 폭풍해일, 월파, 강우, 하천유량, 내수배제 불량 등 둘 이상의 침수 요인이 동시에 또는 연쇄적으로 작용하여 발생하는 침수 현상 |
| 쉽게 이해하기 | 바닷물이 높아지는 상황에서 큰 파도가 방파제를 넘고, 동시에 비가 많이 와서 하천과 배수시설이 감당하지 못하면 침수가 더 커진다. 이렇게 여러 원인이 겹쳐 생기는 침수를 복합침수라고 한다. |
| 과업에서 보는 관점 | 최근 연안재해는 단일 요인보다 복합 요인으로 발생하는 경우가 많기 때문에, 본 사업은 폭풍해일만 따로 보는 것이 아니라 여러 외력조건을 함께 고려해야 한다. |
| 실무 체크포인트 | 어떤 지역에 폭풍해일, 월파, 강우, 하천유량 중 무엇을 조합할지 발주기관과 협의해야 하며, 시나리오별 결과 중첩 방식도 명확해야 한다. |

## 4.3 ADCIRC

| 구분 | 설명 |
|---|---|
| 전문 정의 | Advanced CIRCulation Model의 약자로, 비정형 격자 기반의 조석, 폭풍해일, 연안 순환, 해수면 변동을 모의하는 연안·해양 수치모형 |
| 쉽게 이해하기 | 태풍이 접근할 때 바닷물이 어느 해역에서 얼마나 높아지는지 컴퓨터로 계산하는 모델이다. 복잡한 해안선과 수심을 반영해 연안 수위를 계산할 수 있다. |
| 과업에서 보는 관점 | 해안침수예상도에서 폭풍해일 수위를 산정하는 핵심 모형이다. 전차사업과의 일관성, 검증자료, 계산격자, 수심자료 품질이 중요하다. |
| 실무 체크포인트 | 계산영역, 격자해상도, 수심자료, 조석 검증, 태풍 검증, 관측값과 계산값 비교 결과를 확인해야 한다. |

## 4.4 EurOtop 2018

| 구분 | 설명 |
|---|---|
| 전문 정의 | 유럽의 해안 구조물 월파 산정 지침으로, 파랑 조건과 구조물 형상 정보를 이용하여 평균 월파량을 산정하는 경험식 기반 방법론 |
| 쉽게 이해하기 | 파도가 방파제나 호안을 얼마나 넘어오는지를 계산하는 기준서라고 보면 된다. 파도 높이, 주기, 구조물 경사, 마루높이 같은 조건을 이용한다. |
| 과업에서 보는 관점 | 폭풍해일 수위만으로는 설명되지 않는 침수를 계산하기 위해 월파량 산정이 필요하다. 특히 해안 구조물 뒤쪽 침수위험을 평가할 때 중요하다. |
| 실무 체크포인트 | 구조물 형상자료 확보 여부, 적용 가능한 구조물 유형, 경험식 적용 한계, 계산시간, 기존 경험식과 신규 산정식 비교 여부를 확인해야 한다. |

## 4.5 DEM / LiDAR

| 구분 | 설명 |
|---|---|
| 전문 정의 | DEM은 지표면 고도값을 격자 형태로 표현한 수치표고모델이고, LiDAR는 레이저를 이용해 지형과 구조물의 3차원 높이를 정밀 측정하는 원격탐사 기술이다. |
| 쉽게 이해하기 | DEM은 땅의 높낮이를 숫자로 만든 지도이고, LiDAR는 그 높낮이를 아주 정밀하게 재는 장비와 기술이다. 침수는 낮은 곳부터 발생하므로 지형 높이 자료가 매우 중요하다. |
| 과업에서 보는 관점 | 침수심은 수위와 지형고의 차이로 산정되기 때문에, 지형자료가 틀리면 침수범위와 침수심도 달라진다. |
| 실무 체크포인트 | 최신성, 해상도, 수직기준면, 좌표계, 해안선 및 인공구조물 반영 여부를 확인해야 한다. |

# 6. AI 해양영상 융복합 분석 - 상세 용어 해설

## 6.1 멀티모달 AI

| 구분 | 설명 |
|---|---|
| 전문 정의 | 영상, 레이더, 위치, 센서, 텍스트 등 서로 다른 형태의 데이터를 함께 학습·추론하여 단일 데이터 기반 분석보다 높은 상황인식 성능을 확보하는 AI 기술 |
| 쉽게 이해하기 | 사람은 사진만 보는 것이 아니라 소리, 위치, 시간, 주변 상황을 함께 보고 판단한다. 멀티모달 AI도 카메라 영상, 레이더, 센서, 위치정보를 함께 보고 바다 상황을 판단한다. |
| 과업에서 보는 관점 | 해양 현장은 안개, 야간, 악천후 등으로 영상만으로 판단하기 어려운 경우가 많다. 따라서 여러 데이터를 결합해 객체와 위험상황을 더 정확하게 판단해야 한다. |
| 실무 체크포인트 | 데이터 시간동기화, 좌표 정합, 센서별 품질 차이, 결측 처리, 실시간 처리속도, 플랫폼 연계 인터페이스를 확인해야 한다. |

## 6.2 온디바이스 AI

| 구분 | 설명 |
|---|---|
| 전문 정의 | 클라우드나 중앙서버로 데이터를 전송하지 않고 카메라, 레이더, 엣지 장비 등 현장 장비 자체에서 AI 추론을 수행하는 기술 |
| 쉽게 이해하기 | 인터넷이나 서버 연결이 없어도 현장 장비 안에서 바로 AI가 판단하는 방식이다. 예를 들어 해안 감시카메라 장비 자체가 선박이나 조난자를 즉시 탐지하는 개념이다. |
| 과업에서 보는 관점 | 해양 현장은 통신이 불안정하거나 보안이 중요한 경우가 많다. 온디바이스 AI는 지연시간을 줄이고 보안을 높이며 현장 즉시 대응을 가능하게 한다. |
| 실무 체크포인트 | 모델 경량화, 장비 성능, 전력 소모, 발열, 현장 내구성, 업데이트 방식, 보안정책을 함께 검토해야 한다. |

## 6.3 객체 탐지와 객체 식별

| 구분 | 설명 |
|---|---|
| 전문 정의 | 객체 탐지는 영상이나 센서 데이터에서 관심 객체의 위치를 찾아내는 기술이고, 객체 식별은 탐지된 객체의 종류와 속성, 위험도를 판단하는 기술이다. |
| 쉽게 이해하기 | 객체 탐지는 “화면에 무언가 있다”를 찾는 단계이고, 객체 식별은 “그것이 선박인지, 드론인지, 사람인지”를 구분하는 단계이다. |
| 과업에서 보는 관점 | 탐지만으로는 대응이 어렵다. 실제 운영에서는 어떤 객체인지, 위험한 상황인지, 어느 기관이 대응해야 하는지까지 이어져야 한다. |
| 실무 체크포인트 | 탐지 정확도, 오탐/미탐, 객체 유형 분류 기준, 위험도 판단 기준, 현장 검증 데이터셋 확보가 중요하다. |
