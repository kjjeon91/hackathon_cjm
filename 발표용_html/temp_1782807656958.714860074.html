RFP Ocean Intelligence Before & After

업로드된 RFP Ocean Intelligence v13 - 4개 파일 반영를 기준으로, 기존 수동 문서 확인 방식이 원문 기반 통합 대시보드로 어떻게 개선되는지 정량·정성적으로 정리했습니다.

1. 기존 업무의 문제

Before 수동 확인

  • 사업별 제안요청서를 각각 열람
  • 사업명, 예산, 기간, 발주처를 직접 정리
  • 긴 과업내용에서 핵심 요구사항을 수동 추출
  • 전문용어를 별도로 검색하거나 담당자 경험에 의존
  • 사업 간 비교가 어렵고 정리 형식이 달라짐

After 통합 대시보드

  • 4개 파일을 하나의 목록으로 통합
  • 프로젝트명, 기간, 발주처를 표준화된 표로 제공
  • 사업요약, 대상범위, 산출물, 중요정보를 구조화
  • 상세 용어해설을 과업 관점까지 제공
  • 원문 전체를 연결하여 근거 확인 가능

3. 정량적 개선 효과

4개 파일 통합해안침수, 해양위성, 생태계, 민군경 AI RFP를 하나의 화면에서 확인
3단계 화면 구조목록 → 요약/용어 → 원본 전체 보기로 업무 흐름 표준화
70~80% 절감 예상파일 탐색, 핵심내용 정리, 용어 확인에 드는 반복시간 감소
4개 핵심정보 묶음사업요약, 대상지역/범위, 주요 산출물, 중요정보로 정리
3단계 용어해설전문 정의, 쉬운 설명, 과업 관점으로 해석 기준 통일
LLM 첫 화면 연동AI 관련 사업, 발주처별 사업, 위성 관련 사업 등 빠른 질의 가능

4. 정성적 개선 효과

영역개선 내용업무 효과
일관성모든 사업을 같은 화면 구조와 같은 정보 단위로 정리담당자별 정리 방식 차이를 줄이고 비교 가능성 향상
이해도전문용어를 전문정의·쉬운 설명·과업 관점으로 분리비전문자도 과업 핵심을 빠르게 이해 가능
근거성요약과 원문 전체 보기를 함께 제공요약 신뢰성을 높이고 원문 확인 시간을 단축
탐색성목록과 LLM 질문을 첫 화면에 배치관심 사업을 빠르게 찾고 질문 기반 탐색 가능
확장성동일 구조로 새로운 RFP 파일을 추가 가능향후 수주검토·위험평가·결재보고서 자동화와 연계 가능
