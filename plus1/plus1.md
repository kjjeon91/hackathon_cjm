# RFP Decision Intelligence v9 설계 메모

## 변경사항

v5에서는 자동 결과보고서 샘플이 플랫폼 내부 6페이지에 포함되어 있었다.  
v7에서는 부서장 판단 페이지의 사업부서 예산을 직접 입력하지 않고 **전체 사업 예산 × 사업부서 예산 비율**로 자동 산정하도록 변경하였다.

---

## 핵심 구조

```text
1. 플랫폼 화면
- 파일 업로드
- 사업개요 확인
- 부서장 판단
- 행정부 판단
- 부문장 판단
- 결과보고서 생성 설정

2. 별도 결과보고서 파일
- 사업 개요
- 주요 사업내용
- 역할별 검토 결과
- 주요 리스크 및 대응방향
- 종합 판단
- 결재란
```

---

## 6페이지 변경

### 기존 v5
```text
6페이지 화면 안에 자동 결과보고서 본문이 직접 표시됨
```

### 수정 v6
```text
6페이지는 결과보고서 생성 설정 화면으로만 사용
결과보고서 본문은 별도 HTML/DOCX/HWP/PDF 파일로 생성
```

---

## 프로토타입 제공 파일

```text
- 플랫폼 HTML v6
- 자동 결과보고서 샘플 HTML v6
- MD 설계서 v6
```

---

## 실제 구현 시

```text
검토값 저장
→ 보고서 템플릿에 자동 삽입
→ DOCX/HWP/PDF 생성
→ 그룹웨어 결재 API로 자동 상신
```


---

## v7 추가 변경사항

### 1. 부서장 판단 페이지 예산 연동

부서장 판단 페이지에서 다음 구조로 변경한다.

```text
전체 사업 예산 입력
사업부서 예산 비율(%) 입력
→ 사업부서 예산 자동 산정
```

예시:

```text
전체 사업 예산: 15,000,000,000원
사업부서 예산 비율: 7.97%
사업부서 예산: 1,195,500,000원
```

### 2. 샘플 결과보고서 별도 HTML 파일 유지

플랫폼 파일 안에 결과보고서 본문을 넣지 않고,
별도 결과보고서 HTML 파일을 함께 제공한다.

제공 파일:

```text
1. 플랫폼 HTML
2. 샘플 자동 결과보고서 HTML
```

실제 구현 시에는 별도 결과보고서를 HTML뿐 아니라 DOCX, HWP, PDF로도 생성하도록 확장한다.

## v8 추가 변경사항

### 종합평가 결과 요약 추가

최종 페이지의 **종합 평가 레이더 차트 아래**에 자동 요약 영역을 추가한다.

표시 내용:

```text
- 종합 참여 판단
- 가장 강한 평가축
- 보완이 필요한 평가축
- 역할별 평균점수 요약
- 우선 보완 대상
```

### 별도 샘플 결과보고서에도 종합평가 요약 반영

별도 생성되는 샘플 결과보고서 HTML에도 다음 섹션을 추가한다.

```text
5. 종합평가 결과 요약
- 기술수행
- 행정안정
- 전략가치
- 기관위험
```

## v9 추가 변경사항

### 1. 첫 페이지 평가자 지정 영역 추가

공고문 업로드 아래에 평가자 및 결재 검토자를 지정하는 영역을 추가한다.

기본값:

```text
부서장 / 최초 평가자: 최흥배 부서장
행정부 검토자: 오주혜 선임
부문장 검토자: 송용식 부사장
```

업무 흐름:

```text
부서장 평가
→ 지정된 행정부 평가 요청
→ 지정된 부문장 평가 요청
→ 자동 결과보고서 생성
→ 결재 상신
```

### 2. 종합평가 결과 요약 표시

최종 페이지의 종합 평가 레이더 차트 아래에 다음 내용을 표시한다.

```text
- 종합 참여 판단
- 가장 강한 평가축
- 보완이 필요한 평가축
- 역할별 평균점수 요약
- 우선 보완 대상
```

### 3. 샘플 결과보고서 레이더 차트 포함

별도 생성되는 샘플 결과보고서 HTML에 다음 차트를 포함한다.

```text
- 부서장 레이더 차트
- 행정부 레이더 차트
- 부문장 레이더 차트
- 종합 평가 레이더 차트
```
